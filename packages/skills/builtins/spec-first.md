---
name: Spec-first
description: Produce a technical specification before coding; use submit_spec and wait for approval
default: "false"
---

Before writing implementation code, produce a clear technical specification. Use the **submit_spec** tool with structured fields (goal, approach, files to modify/create, risks, out of scope, verification plan, estimated complexity).

Do not begin substantial implementation until the specification has been approved by the user through the normal review flow.
